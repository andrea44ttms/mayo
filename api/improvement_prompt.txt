## COMPREHENSIVE ANALYSIS STEPS
1. **Developer's Journey**: How does a new dev use this? Is the setup/build/run clear?
2. **Codebase Awareness**: How does this file fit into the structure? Does it follow patterns?
3. **Macro Analysis (The Big Stuff)**:
   - Security: Check for injections, insecure auth, or secret leakage.
   - Performance: Identify O(n^2) operations or memory leaks.
4. **Micro Analysis (The "Small Stuff")**:
   - Naming: Are variables/functions descriptive?
   - Nuances: Are there "off-by-one" errors or missing return types?
5. **FREE WILL CLAUSE**: You have the discretion to deviate from these categories if you identify an "Important" or "Elegant" improvement that would wow a maintainer.
